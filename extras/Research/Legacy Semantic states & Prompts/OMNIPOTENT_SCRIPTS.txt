⚔️ OMNIPOTENT SCRIPT FRAMEWORK — commander input → soldier output
A golden, repeatable pattern for producing zero-damage, self-verifying, idempotent scripts that turn ideas into working systems with minimal back-and-forth.

2) Golden Phases (what every script does)

Preflight Probe (Read-Only)

Environment: users, groups, shells, sudo, PATH

Filesystems: binds & RO (findmnt), free space (df -h)

Permissions: test -w, ACLs (getfacl), ownership sanity

Dependencies: presence + versions (command -v, dpkg -s)

Resources: CPU (nproc), mem (/proc/meminfo), ports (ss -lntp)

App state: processes (pgrep -fa …), service status/log hints

Outputs: GO/NO-GO checklist + remediation hints

Dry-Run Plan

Prints intended actions with targets (paths, owners, modes, diffs)

Simulated effects with no writes (and what would cause rollback)

Backup (Atomic)

Timestamped copy of every target file/dir into a versioned tree

Verifies readability & checksums

Apply (Minimal-Change)

Write only what’s needed, prefer additive changes

Never write to RO mounts—write to admin source behind the bind

Self-Test (Prove It)

Before/after deltas (ports, PIDs, HTTP 200s, tail of logs, file perms)

Sanity assertions on expected strings / endpoints / artifacts

Report & Evidence

Single TXT on Commander’s Desktop: probes, actions, results, PASS/FAIL

Quick remediation section

Rollback (If Any Core Check Fails)

Automatic restore of touched files from the backup set

Re-verify; record rollback results

Idempotency Guard

Re-runs are safe (detect & skip identical state)

4) Dry-Run Gates (Always enforce)

Writable admin source behind any RO bind (e.g., /etc/pin → /srv/pin_admin/config)

Backup root writable

Log path creatable + owned by runtime user

Dependencies resolvable (or queued to install in execute phase)

Core artifacts present (or will be created) with precise paths

No destructive action without a valid backup in place

Dry-run = FAIL ⇒ hard NO-GO with a clear report.
Dry-run = PASS ⇒ 7-second cancel gate ⇒ execute.

5) Resource & Dependency Strategy

Auto caps: compute threads = ceil(nproc * pct); never <1

Think-time: pass through to runtime or UI fallback delay (never hard-block UI)

Model auto: prefer current.gguf → else freshest/first *.gguf

Deps: minimal footprint, --no-install-recommends, tolerate offline apt by degrading checks (warn, don’t crash)

6) Safety Invariants

Zero-damage by default (no changes if any guard fails)

No surprises (no auto restarts unless explicitly commanded)

Least privilege (writes only where necessary; never to RO)

Evidence first (prove success with concrete probes, not assumptions)

7) Quality Scorecard (for any script  made)

I’ll grade each deliverable (0–5, target ≥4 in every row; <4 ⇒ iterate):

Clarity (structure, comments, readability)

Safety (backups, rollback, RO awareness, no-damage)

Idempotency (re-run behavior clean and predictable)

Observability (logs, report, PASS/FAIL matrix)

Determinism (pinpointed paths/owners/modes, no fuzzy matches)

Probe Coverage (env, deps, resources, ports, logs)

User Control (Commander can cancel, start/stop manually)

Portability (few distro assumptions; guarded apt use)

I’ll include this score in the report and call out any sub-4 items with fixes.

8) Soldier Template (skeleton every heavy script will follow)

#!/usr/bin/env bash set -Eeuo pipefail # === CONFIG === PIN_USER="${PIN_USER:-pin}"; LINK_USER="${LINK_USER:-link}" ADMIN_CONF_DEFAULT="/srv/pin_admin/config"; CONF_TARGET="/etc/app" ADMIN_ROOT="/usr/local/admin"; BACKUP_ROOT="$ADMIN_ROOT/backups" REPORT="$HOME/Desktop/<proj>_report_$(date +%F_%H-%M-%S).txt" # === UTILS === say(){ echo "$*"; }; hr(){ printf '%*s\n' 80 | tr ' ' '-'; } ensure_root(){ [ "$(id -u)" -eq 0 ] || { echo "sudo bash $0"; exit 1; } } ensure_dirs(){ mkdir -p "$@" || true; } backup_file(){ [ -e "$1" ] && { mkdir -p "$BACKUP_DIR$(dirname "$1")"; cp -a "$1" "$BACKUP_DIR$1"; }; } # === PREFLIGHT (READ-ONLY) === ensure_root : > "$REPORT"; exec > >(tee -a "$REPORT") 2>&1 # detect admin source behind bind ADMIN_CONF_SRC="$(findmnt -no SOURCE,OPTIONS --target "$CONF_TARGET" 2>/dev/null | awk '/bind/ {print $1}')" [ -z "$ADMIN_CONF_SRC" ] && ADMIN_CONF_SRC="$ADMIN_CONF_DEFAULT" DRY_FAIL=0 # …probes here… set DRY_FAIL>0 on block if [ "$DRY_FAIL" -gt 0 ]; then echo "DRY-RUN FAIL"; exit 10; fi echo "DRY-RUN PASS; executing in 7s"; sleep 7 # === BACKUP === BACKUP_DIR="$BACKUP_ROOT/$(date +%F_%H-%M-%S)" # backup_file "/path/to/file1"; backup_file "/path/to/file2" # === APPLY (MINIMAL) === # write only changed files; avoid RO targets; set owners/modes # === SELF-TEST === # curl/ss/pgrep/tail checks → PASS/FAIL # === ROLLBACK (if fail) === # if fail: cp -a "$BACKUP_DIR/*" / echo "DONE; report at $REPORT" 

9) How to Auto-Adhere (going forward)

Always:

Build preflight probes and a dry-run gate

Detect RO binds and write to the admin source

Create atomic backups and rollback on failure

Produce a Desktop report with PASS/FAIL matrix

Include self-tests that mirror your acceptance criteria

Score the deliverable with the Quality Scorecard

Only ask questions when:

A path or port is ambiguous and cannot be inferred safely

A destructive action is requested without backups/rollback

Security boundaries are undefined (e.g., who may control what)
