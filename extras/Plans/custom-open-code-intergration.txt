# ONF — Full GUI Integration of Open-Code (Rich/TUI → Tk GUI)

## **State**

* The **custom-open-code** build is a **Rich-based TUI** built for terminal interaction:

  * `interactive.py` drives menus, progress, and live console feedback.
  * Core logic lives in “controller” functions that rely on `rich.Console` for rendering.
  * It’s currently **not GUI aware** — no event dispatch, no widget I/O, no non-blocking UX.
* The **Trainer GUI (Tk)** already has:

  * Multi-tab Notebook system.
  * Separate Settings, Models, Training, Evaluation, and Logging subsystems.
  * JSON-based persistent config, model registry, and semantic state system.
* Goal: unify both into a **modular GUI-first framework**, with Open-Code becoming **the GUI's “Toolchain” layer**, not a separate TUI runtime.

# **Phase Breakdown**

### **Phase 0 – Audit & Freeze**

* Locate all `from rich import ...` and `Console()` calls.
* Trace dependencies on `.print()`, `.status()`, `.live()`.
* Freeze current TUI branch (tag as `v-cli-stable`).
  **Deliverable:** `obsolete_map.json` listing all modules with Rich dependency lines.

---

### **Phase 1 – Decouple Core Logic**

* Move all operational functions (tools, file ops, benchmarks) into **pure Python modules** (`open_code_core/`).
* Replace `console.print` calls with `self._emit(event, payload)`.
* Introduce a small **event bus** system:

  ```python
  from Data.utils.gui_event_bus import emit
  emit("progress_update", {"pct": 42, "msg": "Benchmark running"})
  ```

---

### **Phase 2 – GUI Controller Layer**

* Build Tk tabs mirroring previous menu hierarchy:

  * Open-Code (main)
  * Tools
  * Benchmarks
  * Repos
* Use `ttk.Notebook` sub-tabs for nested menus.
* Add event bindings for progress updates and log messages.

---

### **Phase 3 – Integration with Trainer**

* Connect to shared:

  * Settings (via `config.py`)
  * Logger (unified `Data/logger.py`)
  * Model registry (`Models/`)
  * Eval engine (reuse `evaluation_engine.py` for benchmark calls)
* Replace CLI file management with calls to Trainer’s storage APIs.

---

### **Phase 4 – Full GUI Control Features**

* Replace `rich` progress bars with `ttk.Progressbar`.
* Replace live logs with Tk `Text` widget + `Scrollbar`.
* Replace menu text prompts with dropdowns, comboboxes, and toggles.
* Add persistent layout state (window sizes, tab focus) to `settings.json`.

---

### **Phase 5 – Retirement**

* Deprecate or delete:

  * `interactive.py`
  * `rich_console.py`
  * CLI configs/themes
* Update documentation:

  * `README_GUI.md`
  * Developer migration guide
  * Change log summary

---

# **Problem Card**

**Problem:** TUI interface limits user control and integration with Trainer GUI.

**Done when:**
