### 3) Type detection (rules + signals)

Create `model_typing.py`:

```python
from pathlib import Path
from typing import Dict, List

TYPE_RULES = [
  {"type":"Coder", "keywords":["code_analyze","git_operations","Qwen2.5-Coder","Coder"], "categories":["Tools","Git","Code"]},
  {"type":"Researcher","keywords":["web_search","web_fetch","research"],"categories":["Web"]},
  {"type":"Image-Processor","keywords":["image","vision","clip"],"categories":["Image"]},
  {"type":"Intent-Classifier","keywords":["classifier","intent"],"categories":["Semantic_States"]},
  {"type":"Orchestrator","keywords":["process_manage","bash_execute","auto-chain"],"categories":["Multi_Tool","Orchestration"]},
]

def infer_type(signals: Dict[str,List[str]]) -> Dict:
    s = " ".join(sum(signals.values(), []))
    # tally by rule match
    best, conf = "Generalist", 0.0
    for rule in TYPE_RULES:
        hits = sum(k in s for k in rule["keywords"])
        cat_hits = sum(c in s for c in signals.get("categories",[]))
        score = hits*0.7 + cat_hits*0.3
        if score>conf:
            best, conf = rule["type"], min(1.0, score/3.0)
    return {"label":best,"confidence":conf,"signals":signals}
```

**Signals we feed in:**

* training categories used (from dataset paths), tool names seen in schemas, model name, profile tags, and any selected **system prompt / tool schema** used during Evaluation (your UI plan mentions selecting these).

### 4) Evaluation integration (score → rank)

Extend `evaluation_engine.py` (from your plan) to produce atomic sub-scores and a composite:

```python
def compute_caps(eval_metrics)->dict:
    # map your test suites to caps
    return {
      "coherence": eval_metrics["chat_coherence_acc"],
      "tool_basic": eval_metrics["exact_match@1_tools_basic"],
      "single_step": eval_metrics["single_step_pass"],
      "tool_mastery": eval_metrics["multi_arg_tool_f1"],
      "multi_step": eval_metrics["multi_step_chain_success"],
      "reasoning": eval_metrics["llm_judge_reasoning>=good"],
      "creativity": eval_metrics["divergent_solutions_rate"],
      "efficiency_tokens": 1.0 - eval_metrics["avg_tokens_norm"],  # invert
      "efficiency_latency": 1.0 - eval_metrics["avg_latency_norm"]
    }

def rank_from_caps(caps, weights, ranks):
    score = sum(caps[k]*weights[k] for k in weights)
    eligible = [r for r in ranks if score>=r["min_score"]]
    tier = eligible[-1] if eligible else ranks[0]
    return {"tier": tier["tier"], "title": tier["name"], "score": round(score,3)}
```

Hook this right after you aggregate test results (fits perfectly with your “run_benchmark / post-training eval” flow).

## Problem Card

* **Challenge:** Auto-classify models and award “fun but meaningful” ranks that reflect capability and resource behavior—without adding heavy ops overhead.
* **Constraints:** Offline/Local friendly; plug into existing tabs; leverage current stats/logs.
* **Success:** UI shows badges; ranks track eval gains; types look “right”; Commander can sort & compare.

## Option Matrix (type detection approaches)

| Option                     | How                                      | Pros                        | Cons                        | Pick?                   |
| -------------------------- | ---------------------------------------- | --------------------------- | --------------------------- | ----------------------- |
| Rules from schemas/prompts | Parse tool names, categories, model name | Simple, deterministic, fast | May mislabel hybrids        | **Yes (base)**          |
| Behavior-based             | Infer from eval strengths                | Reflects reality            | Needs good coverage         | **Yes (secondary)**     |
| LLM classifier             | Ask a judge model                        | Flexible                    | Extra token cost + variance | Later                   |
| Manual tag                 | User sets type                           | Accurate override           | Manual work                 | **Yes (override only)** |

**Decision:** Hybrid: rules + behavior; manual override allowed.

## Red-Team

* **Metric gaming:** Add adversarial evals + periodic held-out checks (your plan already flags this).
* **Bias from prompts:** Keep “type” confidence score; show signals; allow override.
* **Over-promising tiers:** Keep thresholds configurable; surface next-tier deltas.

## Decision Sheet

* **Go with** hybrid type detection + weighted rank score; wire into Evaluation engine; persist to `skills/`.
* **UI**: badges + next-tier targets.
* **Policy**: default thresholds above; Commander can tune in Settings.

## Quick code you can paste today

**config.py** (helpers)

```python
from datetime import datetime
from pathlib import Path
import json

from .rank_config import RANKS, RANK_COLORS, RANK_WEIGHTS  # split if you prefer

SKILLS_DIR = Path("skills"); SKILLS_DIR.mkdir(exist_ok=True)

def save_skills_profile(model_name:str, profile:dict):
    p = SKILLS_DIR / f"{model_name}.json"
    profile["awarded_at"] = datetime.utcnow().isoformat()
    p.write_text(json.dumps(profile, indent=2))

def load_skills_profile(model_name:str)->dict|None:
    p = SKILLS_DIR / f"{model_name}.json"
    return json.loads(p.read_text()) if p.exists() else None
```

**evaluation_engine.py** (merge with your pipeline)

```python
from config import save_skills_profile, RANKS, RANK_WEIGHTS
from tools.model_typing import infer_type

def finalize_gamified_profile(model_name, eval_metrics, signals):
    caps = compute_caps(eval_metrics)         # as defined above
    rank = rank_from_caps(caps, RANK_WEIGHTS, RANKS)
    mtype = infer_type(signals)
    save_skills_profile(model_name, {

        "rank": rank,
        "caps": caps,
        "evidence": {"eval_run_id": eval_metrics["run_id"], "benchmark_delta": eval_metrics.get("delta",{})}
    })
    return {"type": mtype, "rank": rank, "caps": caps}
```
