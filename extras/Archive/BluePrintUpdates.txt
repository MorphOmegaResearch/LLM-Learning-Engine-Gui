    🎯 Key Patterns to Document

    Pattern 1: Tab + Subtabs + Sidebar

    Most tabs use this structure:
    - Left: ttk.Notebook with subtabs
    - Right: Context-sensitive sidebar menu
    - Header: Title + Refresh button

    Pattern 2: BaseTab Safe Loading

    All tabs inherit from BaseTab:
    - safe_create() wraps create_ui()
    - Error isolation prevents crashes
    - Retry button on failure

    Pattern 3: Master Refresh System

    - Each tab: refresh_<tab>_tab() method
    - Settings: refresh_all_tabs() master method
    - Passes main_gui reference to Settings for cross-tab access

    Pattern 4: Model Display Syncing

    - Config panel selection triggers Runner update
    - Tab change events trigger display updates
    - Color-coded visual feedback
