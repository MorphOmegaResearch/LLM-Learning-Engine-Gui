  5. Component Specifications

  (See v1.7a for components not mentioned here. This version focuses on Models Tab upgrades.)

  5.4. `tabs/models_tab/models_tab.py` (Models Tab)
   - Right-side listing
     - Two scrollable sections:
       - Base Models (PyTorch): Each base row has a left arrow to expand/collapse nested Levels beneath it.
       - Ollama Models (GGUF): Grouped by base (when discoverable via Level export manifests) with their own per-group expand/collapse; unassigned models appear under “Unassigned”.
     - Dynamic grouping: `_group_ollama_by_base()` maps Ollama export names back to their base via discovered Levels; falls back to “Unassigned”.

   - Levels workflow
     - Levels appear nested under their base in the Base Models section.
     - Selecting a Level shows its overview (base/adapters/evaluation summary/status/actions) in the left pane.
     - “Export to GGUF” on a Level triggers the unified export dialog (see below).

   - Unified Confirm & Export dialog (Conformer)
     - Single dialog used by both Adapters → Level Up and History → Levels → Export to GGUF.
     - Shows:
       - Adapter name, readiness status, and Overall Δ vs baseline (when available), plus issues list.
       - Quantization dropdown: `q4_k_m`, `q5_k_m`, `q8_0`.
       - Live estimates, per selected quant:
         - Estimated GGUF file size (heuristic from HF base size × quant ratio).
         - Runtime RAM (load) ≈ 1.2× file size.
         - Disk free recommended during export ≈ 2× file size.
         - Estimated tokens/sec (CPU, ~N threads) as a range, scaled by cores and quant.
         - Quality notes per quant.
     - Export passes `--quant` to the exporter and streams logs to the Evaluation output console.
     - Exporter path resolution fixed to `Data/merge_and_export.py`.

   - History → Runs enhancements
     - Each adapter entry shows latest evaluation summary and actions:
       - View Report (opens latest evaluation JSON).
       - Compare (vs baseline with Δ%)
       - Open Folder (adapter directory)

   - Other
     - Nested expand/collapse UI state tracked per base/group.
     - Styling: small, black text in the dialog’s info/estimates for clarity.

  6. Data Flow & State
   - `ModelsTab` state additions:
     - `levels_expanded`: map of base → bool for Levels expansion.
     - `ollama_expanded`: map of base → bool for Ollama group expansion.
     - `ollama_ui`: pointers to per-group arrow/button containers.
   - Export flow:
     - Readiness computed -> unified dialog -> `_run_merge_and_export(base, adapter, quant)` -> background thread streams output.
