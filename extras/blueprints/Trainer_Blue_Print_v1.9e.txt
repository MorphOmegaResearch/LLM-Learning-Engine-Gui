  🏗️ OpenCode Trainer System - Complete Blueprint (v1.9e - Chat History Overhaul)

  Highlights in v1.9e - Chat History Overhaul
  - Per-Message Saving: Chat history is now saved after every model response, ensuring no data is lost.
  - Renewable Sessions: The "Clear Chat" button now saves the current conversation before starting a new session, preventing accidental data loss.
  - Functional History Tab:
    - The "History" sub-tab now correctly loads and displays all saved chat sessions from disk.
    - The list of sessions automatically refreshes when a new conversation is saved.
  - Session Configuration Display:
    - A "View Config" button was added to each session in the "History" tab.
    - This displays all relevant session metadata: mode, temperature, system prompt, tool schema, working directory, training mode status, model, and tool settings.
  - Comprehensive Metadata: Sessions now store a complete snapshot of the configuration active during the conversation.

  Highlights in v1.9d - Temperature Controls
  - Global Temperature Control:
    - Added a temperature scale to the "Basic" settings tab.
    - Allows setting a global temperature for all models.
    - The setting is persisted in `custom_code_settings.json`.
  - Session-Specific Temperature Control:
    - Added a temporary temperature scale to the chat interface.
    - Allows overriding the global temperature for the current chat session.
    - Resets to the global temperature when a new model is selected or the chat is cleared.
