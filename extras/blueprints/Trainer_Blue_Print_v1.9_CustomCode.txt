  ## Technical Implementation Details

  ### Chat Interface Flow
  1. User selects model from right panel → `select_model()` called
  2. Model name displayed in chat header, send button enabled
  3. User types message and presses Enter → `send_message()` triggered
  4. Message added to chat display and history
  5. Background thread spawned → `generate_response()` runs
  6. Ollama API called via curl subprocess with JSON payload
  7. Response parsed and displayed, chat history updated
  8. UI buttons reset to ready state

  ### Ollama API Integration
  **Endpoint:** `http://localhost:11434/api/chat`
  **Method:** POST with JSON
  **Payload Structure:**
  ```json
  {
    "model": "model-name",
    "messages": [
      {"role": "user", "content": "user message"},
      {"role": "assistant", "content": "bot response"}
    ],
    "stream": false
  }
  ```
  **Implementation:** Lines 269-285 in chat_interface_tab.py
  - Uses tempfile for payload
  - curl subprocess for HTTP request
  - 120-second timeout
  - Error handling with user feedback

  ### Threading Strategy
  - Main UI thread: tkinter event loop
  - Background thread: Ollama API call (prevents UI freeze)
  - `root.after(0, callback)`: Thread-safe UI updates
  - State management: `is_generating` flag prevents concurrent requests

  ### Style Consistency

  ## Troubleshooting

  ### Tab Not Appearing
  **Symptom:** Custom Code tab enabled but not visible
  **Cause:** Not in tab_order array
  **Fix:**
  1. Check `Data/settings.json`
  2. Verify `"custom_code_tab"` is in `"tab_order"` array
  3. Quick Restart or manual restart

  ### Chat Not Working
  **Symptom:** "Ollama error" when sending message
  **Cause:** Ollama not running or wrong port
  **Fix:**
  1. Check Ollama: `ollama list`
  2. Verify service: `curl http://localhost:11434/api/tags`
  3. Restart Ollama if needed

  ### Import Errors
  **Symptom:** Tab fails to load, error in debug log
  **Cause:** Missing dependencies or path issues
  **Fix:**
  1. Run test script: `python3 test_custom_code_tab.py`
  2. Check debug log for specific import error
  3. Verify file structure matches blueprint

  End-to-End Flow (Updated for v1.9)

  ═══════════════════════════════════════════════════════════════════════

  1) Start with a Base (PyTorch)
     - Select a local base (Models tab, Base Models section)

  2) Pull/Prepare Datasets
     - Place JSONL training/evaluation sets under Training_Data-Sets

  3) Run Pre-Training Baseline
     - Models → 🧪 Evaluation: choose suite + Tool Schema
     - If no GGUF exists, export/pull; evaluation auto-resumes
     - Baseline saved to Models/benchmarks and set active

  4) Train
     - Training → Runner. Commit a model and scripts; start training
     - Post-training evaluation is automatic

  5) Level Up + Export
     - Create Level 1 from adapters
     - Export Level to GGUF with desired quant

  6) Evaluate & Compare
     - Run evaluation for the Level; compare vs parent's baseline
     - Review regression summary and per-skill changes
     - View detailed evaluation in Stats tab
     - Get training suggestions if regressions found

  7) **NEW: Interactive Testing (v1.9)**
     - **Custom Code → Chat**: Test model conversationally
     - Select any Ollama model from right panel
     - Ask questions, verify responses
     - Identify weaknesses for targeted training
