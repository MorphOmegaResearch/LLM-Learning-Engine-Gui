🔧 VERSION CREATION PROCESS:
1. /version new from=stable
2. create_version() copies site-packages, config from STABLE
3. Generates per-version Launcher with correct paths/environment
4. validate_version() checks completeness and writes manifest.json
5. Version becomes selectable if validation passes

🧪 VERSION TESTING PROCESS:
1. /version v1.15
2. interactive.py uses os.execv to replace process
3. Per-version Launcher sets OPENCODE_VERSION=v1.15 environment
4. TUI shows "Version: v1.15 (test)"
5. Auto-return to STABLE on quit/crash via exec (no infinite loops)

🛡️ SAFETY MECHANISMS:
- Per-version crash logs: versions/v#/crash_log.txt
- Manifest validation prevents launching broken versions
- Auto-rollback to STABLE on test failures
- Process replacement (exec) prevents resource leaks
- Debug output for troubleshooting execution paths

OVERVIEW
- Adds experiment-based coding workflow under versions/v1.2.
- Provides /coding commands to create, select, and operate within isolated experiment folders.
- Integrates smart-mode confirmations for time budgets and tool chains.

DIRECTORY LAYOUT (v1.2)
- versions/v1.2/experiments/
  - exp_YYYYMMDD_HHMMSS_name/
    - code/           # Scripts live here; /coding run executes from here
    - output/         # User output area; tool writes safe here
    - README.md       # Experiment metadata
  - .state.json       # Atomic active experiment pointer

COMMANDS
- /coding new [name]              # Create experiment (creates code/, output/, README.md, code/hello.txt)
- /coding list                    # List experiments; marks active
- /coding open <name>             # Set active experiment (confirmation)
- /coding where|status            # Show active experiment (path/name)
- /coding gen <file.py> [text]    # Create/overwrite a script in code/ (confirmation)
- /coding run <file.py>           # Execute script via python inside code/ (confirmation)

BEHAVIOR & SAFETY
- Active experiment working dir: tools default to active experiment path.
- Name sanitization: [A-Za-z0-9_-] (filenames in gen allow .).
- .state.json updates: atomic write (tmp + replace) to prevent corruption.
- Confirmations via confirmation_gate:
  - /coding open: LOW risk — change active pointer
  - /coding gen:  MEDIUM risk — create/overwrite file
  - /coding run:  LOW risk — execute script within experiment
- Smart mode confirmations:
  - /mode thinktime: confirmation before applying time budget
  - Tool chains: risk-based confirmation before execution in smart mode
- Headless/non-TTY: LOW risk auto-confirms via confirmation_gate; no hangs.

QUICKSTART (v1.2)
1) /coding new demo1
2) /coding gen hello.py "print('hello from experiment')"
3) /coding run hello.py
4) /coding where

LLM-DRIVEN WORK (SMART MODE)
- /mode smart
- Ask: “Create a simple TUI app that does <X>.”
- Review tool-chain confirmation summary (risk/time); approve to proceed.
- Generated files appear under active experiment’s code/. Use /coding run to execute.

TROUBLESHOOTING
- If /coding responds but does nothing, ensure:
  - Handler present: hasattr(InteractiveMode,'handle_coding_command') == True
  - OPENCODE_VERSION=v1.2 in environment (per-version Launcher ensures this)
  - PYTHONPATH points to versions/v1.2/site-packages (Launcher sets this)
- Check versions/v1.2/crash_log.txt for tracebacks.

VALIDATION CHECKS
- Compile: python3 -m py_compile versions/v1.2/site-packages/opencode/interactive.py
- State file: cat versions/v1.2/experiments/.state.json (active_experiment updated)
- Filesystem:
  - ls versions/v1.2/experiments/demo1/code/
