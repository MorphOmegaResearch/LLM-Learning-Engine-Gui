```python
# --- add near top of interactive.py ---
import re, json

def _extract_single_line_json(raw: str, obj_type: str=None):
    """Return the first one-line JSON object found that contains 'type'==obj_type (if provided)."""
    for ln in raw.splitlines():
        ln = ln.strip()
        if not (ln.startswith("{") and ln.endswith("}")):
            continue
        try:
            j = json.loads(ln)
            if obj_type is None or j.get("type") == obj_type:
                return j
        except Exception:
            continue
    # relaxed regex fallback
    m = re.search(r"\{.*?\"type\"\s*:\s*\"[^\"]+\".*?\}", raw, flags=re.S)
    if m:
        try:
            j = json.loads(m.group(0))
            if obj_type is None or j.get("type") == obj_type:
                return j
        except Exception:
            return None
    return None

def _extract_single_line_slash_command(raw: str):
    """Return the first single-line slash command like '/mode smart' or '/thinktime set 240'."""
    for ln in raw.splitlines():
        s = ln.strip()
        if s.startswith("/") and " " in s:
            # simple validation: no surrounding prose, single-line command
            return s
    return None
```

# 1) Sensitivity ‚Äî what‚Äôs fragile and what‚Äôs robust

Sensitivity here means: how strictly the runtime will react to model output, and what causes false positives/negatives.

High-sensitivity components (easy to break, tune carefully)

* **Single-line JSON/tool-call detection** ‚Äî if the model emits invalid JSON or extra text on the same line, the runtime will either fail to parse or mis-handle the intent.
  *Mitigation:* enforce single-line JSON in system prompt; add a robust fallback parser that logs malformed output and asks the model to re-emit.
* **Slash-command single-line detection** ‚Äî requires the model to place the slash command on its own line. If it includes commentary on the same line the runtime may treat it as a non-command.
* **Token truncation** ‚Äî if you send huge context, the model‚Äôs tool-call or decision line can be truncated. Reserve tokens for the assistant output.

Medium-sensitivity (less likely to break but still needs policy)

* **Risk confirmation flow** ‚Äî missing confirmation gates can allow undesired ops. Make sure `confirmation_gate` is invoked before HIGH/CRITICAL ops.
* **Project binding** ‚Äî the runtime must correctly resolve `active_project`/experiment paths. A wrong path means file reads fail or read the wrong place.

Low-sensitivity (robust)

* **Heuristics fallback** ‚Äî if the model doesn‚Äôt emit actionable directives, heuristics can propose actions. This is a safe fallback if you prefer conservative behavior.

Suggested conservative defaults to avoid accidental destructive actions:

* Require explicit confirmation for any `ToolRiskLevel` >= MEDIUM (or at least HIGH).
* Require single-line JSON or slash commands strictly; if malformed, ask the model to re-emit.
* Reserve at least 512‚Äì1024 tokens output budget for the assistant so its single-line directive won‚Äôt be truncated.

# 2) Conformers ‚Äî what they are and which to include

Conformers are short, standardized clarification/permission prompts that the runtime asks the user (and/or model) before taking action. They make the flow explicit and auditable.

I recommend these conformer classes (ordered by frequency):

1. **Read Permission (LOW risk)**

   * When model asks to read project files.
   * Example prompt to user:
     `Conformer: I will read files in 'versions/v1.2/experiments/exp_XXXX/site-packages'. Confirm? [y/n]`
   * Auto-allow policy: allow read-only file reads with implicit confirmation for trusted projects.

2. **Clarify Files/Scope (LOW‚ÜíMEDIUM)**

   * When model says ‚Äúread the project‚Äù but doesn‚Äôt specify files.
   * Prompt: `Which files or directories should I inspect? (e.g., site-packages/*.py or src/*.py)`

3. **Execution Permission (MEDIUM)**

   * For file writes, running tests, git operations.
   * Prompt: `This operation will modify or execute code in <path>. Confirm to proceed (type YES to confirm).`

4. **Thinktime Request (INFO / meta-conformer)**

   * When model asks for time to investigate. The runtime maps this into `/thinktime set <#>`.
   * Prompt to user (presented by runtime): `Model requests 240s thinktime to investigate. Grant? [y/n]`
   * If granted, runtime sets budget and the model may run internal steps or tool calls.

5. **High-Risk Confirmation (HIGH / CRITICAL)**

   * For destructive operations: system-level changes, deletion, network access, or credential use.
   * Prompt: `HIGH RISK: This will run "<command>" with risk level HIGH. Type CONFIRM HIGH to proceed.`

6. **Privacy/Data Access (EXPLICIT)**

   * For reading user/private data or contacting external services.
   * Prompt: `This action will access external network or private files. Proceed? [type ALLOW_NETWORK]`

7. **Conformer Chain (multi-step)**

   * For complex CODE tasks: clarifier chain that collects task, target files, tests to run, and confirmation ‚Äî presented as numbered questions.

# 5) Practical tips ‚Äî what to tune for real-world reliability

* **Token reservation**: keep at least 512 tokens free for the assistant's output; if the model often truncates the JSON directive, increase reserved tokens or reduce prior context.
* **Parser tolerance**: accept both one-line JSON and JSON embedded in reply, but if malformed, re-prompt the model automatically (include raw output in the re-prompt).
* **Logging**: log every raw model output and parsed directive to `logs/raw_model_outputs.log` and `logs/tool_calls.log`.
* **Default auto-allow policy**: for convenience, allow `read` operations implicitly for trusted projects but require explicit confirmation for `file_write`, `bash_execute`, `git_operations`.
* **Timeout behavior**: if model requests `thinktime` but user denies, automatically proceed with a FAST response (model should have fallback plan).
* **User override**: allow admin-level `/mode always-ask` vs `/mode autopilot` setting ‚Äî autopilot lets model run within a lower risk threshold without asking each time.

```python
# interactive.py helpers

def _extract_one_line_json(raw: str, obj_type: str = None):
    """Return the first one-line JSON object whose 'type' (if provided) matches."""
    for ln in raw.splitlines():
        ln = ln.strip()
        if not ln:
            continue
        if ln.startswith("{") and ln.endswith("}"):
            try:
                j = json.loads(ln)
                if obj_type is None or j.get("type") == obj_type:
                    return j
            except Exception:
                continue
    # relaxed regex fallback
    m = re.search(r"\{.*?\"type\"\s*:\s*\"[^\"]+\".*?\}", raw, flags=re.S)
    if m:
        try:
            j = json.loads(m.group(0))
            if obj_type is None or j.get("type") == obj_type:
                return j
        except Exception:
            return None
    return None

def _extract_slash_command(raw: str):
    """Return the first single-line slash command (command + args), or None."""
    for ln in raw.splitlines():
        s = ln.strip()
        # accept format: /command arg1 arg2... but line must begin with slash and not contain extra text
        if s.startswith("/") and len(s.split()) >= 1:
            # simple sanity: no other surrounding prose on same line
            return s
    return None
```

```python
import json, re, time, asyncio
from typing import Optional

def _extract_one_line_json(raw: str, obj_type: str = None) -> Optional[dict]:
    for ln in raw.splitlines():
        ln = ln.strip()
        if ln.startswith("{") and ln.endswith("}"):
            try:
                j = json.loads(ln)
                if obj_type is None or j.get("type") == obj_type:
                    return j
            except Exception:
                continue
    m = re.search(r"\{.*?\"type\"\s*:\s*\"[^\"]+\".*?\}", raw, flags=re.S)
    if m:
        try:
            j = json.loads(m.group(0))
            if obj_type is None or j.get("type") == obj_type:
                return j
        except Exception:
            return None
    return None

def _extract_slash_command(raw: str) -> Optional[str]:
    for ln in raw.splitlines():
        s = ln.strip()
        if s.startswith("/") and len(s.split()) >= 1:
            return s
    return None

async def _safe_execute_tool(interactive_self, name: str, args: dict):
    try:
        res = await interactive_self.tool_manager.execute_tool(name, **args)
    except Exception as e:
        interactive_self.logger.exception("Tool execution failed")
        return {"success": False, "output": "", "error": str(e)}
    # normalize
    return {"success": getattr(res, "success", True), "output": getattr(res, "output", "") or "", "error": getattr(res, "error", None)}

async def _call_model_with_tool_result(interactive_self, tool_payload_str: str, user_input: str):
```
